4.- ¿Qué se entiende por framework? ¿Qué compone en Framework OfBiz?

C. Tema: ERP AbanQ
1.- ¿Cuáles son los módulos de AbanQ? (indique una descripción)
Repuesta: 
Las principales funciones del sistema se dividen en módulos, los cuales se instalan sobre el programa básico. Actualmente el software cuenta con los siguientes módulos integrados:
1.	Facturación: permite el funcionamiento de la empresa gestionando clientes, proveedores, stock de productos, emisión de facturas.
2.	Gestión de tesorería: permite llevar una organizada estructura de todas las facturas y conocer el estado en que se encuentran y a quién corresponden rápidamente
3.	Contabilidad financiera: Facilita el control de las finanzas, control de gastos, entrada y salida de dinero, además de emitir informes de facturas emitidas y recibidas. También es de utilidad para proporcionar las pérdidas o ganancias de las pymes en un periodo determinado.
4.	Talleres de automoción: este módulo complementario está diseñado para la gestión de empresas dedicadas a la reparación, venta y mantenimiento de vehículos
5.	Formación: permite proporcionar las relaciones jerárquicas entre empleados y permite un almacenamiento efectivo y una correcta administración de los datos del personal.
6.	Terminal punto de venta: permite gestionar las ventas de la empresa, donde a los artículos se les asigna una tarifa, asociando la facturación de los distintos clientes y proporcionando distintas formas de pago.
7.	Control de producción (MRP):  permite realizar un completo seguimiento a los procesos de fabricación de producción en series, además de proporcionar los costos asociados a la producción. 
8.	Gestión de campaña y acciones clientes (CRM): gestiona todas las funcionalidades relacionadas con clientes y proveedores de la organización, de esta manera coordinar las relaciones con estos, proporcionando información de cada uno de los movimientos asociados.
9.	Gestión de almacenes: Este módulo está relacionado con los temas de stocks, artículos, tarifas, transferencias y datos de compra. Es de suma utilidad para gestionar los movimientos de stocks en el almacén, relacionándose estrechamente con el módulo de facturación y con el de ventas.
10.	Soporte para códigos de barras: permite la introducción y visualización gráfica de códigos de barras de los distintos productos de la empresa.
Cabe destacar que las dos áreas principales del software están disponibles de manera gratuita, estas son Facturación y Contabilidad Financiera.

4.- Para la elección del motor de BD ¿qué criterio usar? ¿alternativas de motores?
Respuesta:
Se proponen los siguientes criterios para la elección de un motor de BD:
1.	Entender los requerimientos actuales pero pensados en el futuro: Se debe entender los requerimientos funcionales y no funcionales y escoger la base de datos que mejor se ajuste a estos requerimientos. Los requerimientos pueden abarcar lo siguiente:
•	Cantidad de datos y tipo de datos (texto, binarios, espacial u otros tipos específicos).
•	Número simultaneo de usuarios.
•	Disponibilidad: cuánto tiempo puede permitir tener de baja la base de datos.
•	Escalabilidad: qué hará cuando la cantidad de datos y el número de usuarios aumente.
•	Seguridad: cuanto necesitará de características como seguridad e encriptación de datos, administración de usuarios, privilegios.
•	Manejo y administración: cuan amigable quiere que sea la administración de la BD.
•	Que otras características quiere obtener de la BD.

2.	Tomar en cuenta experiencias previas: Debe considerarse sus experiencias previas (si es que las posee) y las habilidades desarrolladas trabajando con bases de datos. Elegir una base de datos conocida en la cual se posee experiencia, reduce tiempo al no tener que familiarizarse nuevamente con la BD y también reduce costos en algunos casos como por ejemplo cuando se desea invertir en un curso para aprender el manejo de cierta BD.

D. Tema: GNUCASH
1.- ¿Qué es XML? y ¿cómo lo usa (o implementa*) GNUCAHS?
Respuesta:
- XML es un subconjunto de SGML(Estándar Generalised Mark-up Language),simplificado y adaptado a Internet. XML es un meta-lenguaje que nos permite definir lenguajes de marcado que permite definir etiquetas personalizadas para descripción y organización de datos.
-El mecanismo de almacenamiento principal de GnuCash es en archivo XML. GnuCash interpreta documentos XML utilizando una codificación de caracteres determinado por la configuración. GnuCash 1.9.0 y sus versiones posteriores escribe el documento XML siempre en codificación UTF-8(formato de codificación de caracteres) y también incluye la declaración de codificación apropiada en la declaración de texto XML de apertura.
Obervación (profesor): La respuesta esta bien, fataría agregar un documento XML que use GnuCash. No es neecsario que sea completo, pero al menos una parte para que visualicen como es un documento XML.

2.- ¿Qué es un protocolo? y ¿en que consiste el protocolo OFX?
Respuesta:
-Un protocolo es un conjunto de reglas establecidas por norma o por costumbre, secuencia detallada de un proceso de actuación científica, técnica, etc.
-El protocolo Open Financial Exchange (OFX ) es un protocolo abierto para operaciones financieras en línea para el formato de datos SGML Y XML. Permite que los bancos se conecten directamente a sus clientes sin la necesidad de intermediarios. Incluye operaciones de declaraciones de descarga bancarias y tarjetas de crédito, transferencia de fondos, pagos, pago de facturas, etc. Es sólo para las transacciones que involucran equipos (no incluye teléfono). Se basa en un cliente / servidor de modelo de petición / respuesta donde las peticiones y las respuestas son archivos de texto, basada en SGML. La especificación OFX es sintácticamente similar al HTML, el uso de etiquetas para identificar y definir los datos. El uso de HTTP para conectarse a un servidor de OFX y el uso de comandos HTTP POST para enviar peticiones al servidor OFX.  El mensaje es la unidad de trabajo con el protocolo y se refiere a una solicitud de par requisito / respuesta. Un mensaje puede contener varias transacciones. Cada transacción tiene un sobre de estado que muestra el éxito o el fracaso de una solicitud. Todos los modelos de encriptación pública (tales como SSL y SET) ofrecen flexibilidad en las opciones de seguridad para las instituciones que implementan la OFX.

Observación (profesor): Me parece más bien la respuesta, y como hablan de HTTP, pueden ver con mayor claidad en concepto.
